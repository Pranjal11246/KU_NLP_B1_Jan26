What is a Word Embedding ?
- Dense vector representation of a word that captures meaning

TF-IDF treats words as independent tokens
TF-IDF does not understand:
car = automobile
mobile = electronics
happy = joyful

So we need a way to represent words based on meaning, not just count

Technique for dense numerical vectors:
Word2Vec
- one of the tehcnique in NLP used to represent words as dense numerical vector
- developed by Google in 2013
- it allows machine to understand the semantic relationship between words by positioning similar words closer together in a high-dimensional vector space


Core idea behind Word2Vec
- Words that appear in similar context have similar meanings
- This is called: Distributional Hypothesis

Core Architecture:
- CBOW - Continuous Bag of Words
  - Model that predicts a target word based on its surronding context words. It is fast for frequent words.
- Skip-gram
  - using a single word to precit the surronding context words. It is fast for infrequent words


CBOW: Predict target word from surronding context
Input: I am learning _______ intelligence
Prediction - artificial

Skip-Gram: Predict surronding words from a target word
Input: artificial
Prediction: intelligence, human, machine


Key differences b/w Sparse Matrix and Dense Matrix
1. Data representation
Sparse - Mostly zero values
Dense - Most values are meaningful, non-zero values

2. Memory efficiency
Sparse - Highly memory-efficient for large datasets
Dense - Memory intnsive for large datasets

- developed by Stanford university

Unlike Word2Vec (predictive model), GloVe is a : Count-based + Matrix factorization
It combines:
1. Statistical word co-occurrence
2. Matrix factorization
3. Linear Algebra

Word2Vec learns embeddings by predicting context
But is uses:
- local context window
- doesn't directly use global corpus statistics

Core Idea behine GloVe:
1. It build word-word co-occurrence matrix

RNN - Recurrent Neural Network
Pre-requisite:
1. NLP
2. ANN

Ex: I did not like this movie

Step-1: Initialize Memory
memory = empty
This memory is called hidden state

Step-2: Read first word
Input - "I"
Updates memory
memory = understanding of "I"

Step-3: Read next word
Input - "did"
Now RNN has:
- previous memory = "I"
- current word = "did"

memory = understanding of "I did"

Step-4: Read next word:
Input - "not"
previous memory = "I did"
current word = "not"
new memory = "I did not"


Step-4: Read next word:
Input - "like"
previous memory = "I did not"
current word = "like" - positive sentiment
new memory = "I did not like"

But memory contains "not"
memory = negative sentiment

Algorithm of RNN:
1. Start with empty memory
2. For each word in the sentence:
- Read the word
- Combine it with previous Memory
- update the memory
3. After last word:
- Use memory to make prediction

new memory = previous memory + current word

- Same network repeated across time
- Same weights used at every step
- This is called weights sharing

Summary of RNN:
1. Handle sequences
2. Maintains memory
3. Works for text, speech, time series

Limitation of RNN:
1. Cannot handle long dependencies
2. Slow (sequential processing)

Types of RNN:
1. One-to-One - process single fixed size input to produce single fixed size output
  Example: Image classification (input one image and get one label like "cat" or "dog")
2. One-to-Many - process single input to produce a sequence of output
  Example: Image Captioning (single image input produces a sequence of words)
3. Many-to-One - takes a sequence of input to produce a single output
  Example: Sentiment Analysis, Spam classification
4. Many-to-Many - process multiple inputs to produce multiple outputs
  Example: Language Translator


Key Architecture of RNN:
- Standard RNN often struggle with the "vanishing gradient problem" where they forget information from the distant past. So we have advanced variants to solve this problem using gating mechanism:
- LSTM - Long short term memory: Uses three gates (input, output, forget)
- GRU - Gated Recurrent Unit: Uses two gates (update and reset)
- BiRNN - Bidirectional RNN: process the sequence in both forward and backward direction
- Deep RNN - Stack multiple RNN layers on top of each other


BPTT - Backpropagation Through Time
- To understand BPTT, think of RNN as standard deep network that has been unrolled in time. Because same parameters are used at every step...

LSTM - Long Short Term Memory

Example: I was watching a movie yesterday with my friends and after a long discussion we realized that movie was bad.

Important word = bad
It appears very late RNN may forget it

RNN problem:
Memory gets weaker as sequence grows: Vanishing memory problem

What LSTM does:
- What to remember
- What to forget
- What to output


Three gates of LSTM:
1. Forget gate: decides what to forget from the memory
2. Input gate: decides what to store in memory
3. Output gate: decides what to predict or output memory

Example: "I went to the movie yesterday...the movie was amazing"
Word -> "yesterday" -> may not matter -> forget

Word -> "amazing" -> important sentiment -> input


Sentence- "I did not like the movie"
Step by Step LSTM Algorithm
1. Init memory
memory = empty

2. Read first word 
"I" -> Not important -> weak memory

3. Read next word
"did" -> memory updated slightly

4. Read next word
"not" -> Input gate -> store "not"

5. Read next word
"like" -> positive
"not" -> negative

LSTM combines - final outcome - negative

Internal flow
At each word:
1. decides what to forget
2. decides what to remember
3. update memory
4. decides output
